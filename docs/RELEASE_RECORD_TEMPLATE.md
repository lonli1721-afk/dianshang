# 发布记录模板 / 上线打钩表

本文档用于每次发布时填写，目标是让每次上线都有统一记录、统一检查项和统一结果留存。

使用方法：

1. 每次准备推送到线上前复制一份本模板。
2. 按顺序填写。
3. 未执行项必须写明原因。
4. 若任一阻塞项未通过，不得上线。

## 10. 快速口令版本

若只需要一个极简版上线口令，可按以下顺序快速核对：

1. 改完了没有？
2. 测过了没有？
3. 测试是不是通过了？
4. 提交存档了没有？
5. 备份风险分级和对应备份确认了没有？
6. 数据目录会不会被覆盖？
7. 用户是不是确认可以部署？
8. 上线后 `/health` 和首页是不是正常？
